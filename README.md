## 1. Описание идеи и концепции проекта

Проект **«TrafficVisionCV»** — это программный комплекс на базе алгоритмов компьютерного зрения, предназначенный для автоматического анализа транспортных и пешеходных потоков на городских перекрёстках в режиме реального времени.

*   **Основная концепция:** Использование существующих камер дорожного наблюдения для получения видеопотока, его обработки нейросетевыми моделями и автоматического извлечения структурированных данных о дорожной обстановке. Система реализует подход *«видео как датчик»*, заменяя дорогие физические радары.
*   **Актуальность:** Традиционные светофоры работают по жестко заданным таймерам, что приводит к образованию заторов на пустых улицах. «TrafficVisionCV» собирает данные для адаптивного управления светофорными фазами, что позволяет разгрузить трафик, сократить время в пути и снизить уровень вредных выбросов.

---

## 2. Цели и задачи системы

**Цель проекта:** Разработать работоспособный прототип системы видеоаналитики, способный с высокой точностью распознавать участников дорожного движения, отслеживать направления их перемещения и вычислять метрики загруженности перекрёстка.

Для достижения цели система решает следующие **задачи компьютерного зрения**:
*   **Детектирование объектов (Object Detection):** Обнаружение и локализация (выделение в рамки/Bboxes) транспортных средств и пешеходов на каждом кадре.
*   **Классификация объектов (Object Classification):** Распознавание типов транспорта (легковой автомобиль, грузовик, автобус, мотоцикл/велосипед, спецтранспорт).
*   **Трекинг объектов (Object Tracking):** Присвоение уникального ID каждому объекту и отслеживание его траектории движения между кадрами для точного подсчета без дублирования.
*   **Выделение зон интереса (ROI - Region of Interest):** Настройка виртуальных зон (полос движения, стоп-линий) для локального подсчета машин.

---

## 3. Функциональные возможности

Система выполняет автоматический сбор и обработку следующих метрик:

1.  **Классифицированный подсчет трафика:** Подсчет количества машин каждого типа, прошедших через перекрёсток за определенный промежуток времени.
2.  **Определение направлений движения:** Фиксация траекторий (сколько машин повернуло налево, направо или поехало прямо с конкретной полосы).
3.  **Оценка длины очередей:** Измерение количества автомобилей, стоящих перед стоп-линией в ожидании зеленого сигнала светофора.
4.  **Распознавание приоритетного транспорта:** Выделение в потоке машин экстренных служб (скорая помощь, пожарная служба) для возможности передачи им приоритета проезда.

## 4. Сценарии использования (Use Cases)

> ## Сценарий 1: Адаптивное управление светофорами
> *   **Процесс:** Система в реальном времени анализирует длину очереди на всех подступах к перекрёстку. Данные передаются на дорожный контроллер.
> *   **Результат:** Светофор динамически продлевает зеленую фазу для загруженного направления и сокращает для свободного, минимизируя время простоя.

> ## Сценарий 2: Городская аналитика и планирование
> *   **Процесс:** Программа работает в фоновом режиме, накапливая статистику по часам пик, дням недели и сезонности в базу данных.
> *   **Результат:** Администрация города получает точные отчеты для изменения схем движения, оптимизации маршрутов общественного транспорта или расширения дорог.

> ## Сценарий 3: Безопасность и фиксация инцидентов
> *   **Процесс:** Алгоритм фиксирует аномалии в траекториях (резкая остановка в центре перекрёстка, движение по пешеходному переходу во время запрещающего сигнала).
> *   **Результат:** Оперативное автоматическое уведомление диспетчерских или экстренных служб о возникновении ДТП.

## 5. Предварительный технологический стек

*   **Язык программирования:** Python
*   **Библиотеки CV и ИИ:** OpenCV (предобработка видео), Ultralytics YOLOv8 / YOLOv11 (детектирование и классификация), ByteTrack / BoT-SORT (трекинг объектов).
*   **Интерфейс и визуализация:** Streamlit (для создания интерактивного дашборда с графиками и видеопотоком).

---

## 6. Ожидаемый результат разработки

На выходе планируется получить программный модуль, принимающий на вход видеопоток (файл или RTSP-ссылку с камеры), который выдает:
1.  **Визуализированное видео** с наложенными рамками детектирования, отображением уникальных ID и линий траекторий.
2.  **Аналитическую веб-панель**, на которой отображаются графики текущей загруженности, скорость потока и статистика прошедшего транспорта.
