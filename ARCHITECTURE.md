# Техническая документация проекта «TrafficVisionCV»


## 1. Архитектура видеопайплайна (Pipeline)

Обработка видеопотока в системе «TrafficVisionCV» строится по линейному конвейерному принципу:

[Входной видеопоток] -> [Предобработка кадра] -> [Нейросетевой инференс (Детекция)] -> [Объектный трекинг] -> [Аналитическая логика (ROI)] -> [Визуализация / Дашборд]



## 2. Используемые библиотеки и их техническая роль

### 2.1. OpenCV (opencv-python)
Роль в проекте: Ввод-вывод видеоданных, предобработка изображений и базовая геометрия.
*   Используемые модули и методы:
    *   `cv2.VideoCapture()` — захват видеопотока (чтение локального видеофайла или RTSP-потока с дорожной камеры).
    *   `cv2.resize()` — масштабирование разрешения входного кадра для оптимизации скорости работы нейросети.
    *   `cv2.polyline()` / `cv2.rectangle()` — отрисовка виртуальных зон интереса (полос) и ограничивающих рамок (bounding boxes) вокруг автомобилей.
    *   `cv2.pointPolygonTest()` — ключевой математический метод для проверки, пересекла ли координата автомобиля (трек-поинт) границу полосы движения или стоп-линию.

### 2.2. Ultralytics YOLOv11 (ultralytics)
Роль в проекте: Сверхбыстрое детектирование и классификация объектов в реальном времени.
*   Используемая модель: Предобученная архитектура `yolov11n.pt` (Nano) или `yolov11s.pt` (Small), обученная на датасете COCO, которая «из коробки» умеет распознавать классы: car (легковая), truck (грузовик), bus (автобус), motorcycle (мотоцикл), bicycle (велосипед), person (пешеход).
*   Используемые методы:
    *   `model.predict(source, stream=True)` — запуск инференса в режиме генератора (стриминга) для минимизации потребления оперативной памяти.
    *   Фильтрация по аргументу `classes=[2, 3, 5, 7]` — ограничение детекции только транспортными средствами и пешеходами (исключая лишние классы COCO).

### 2.3. ByteTrack (встроена в пакет ultralytics)
Роль в проекте: Поканальный трекинг объектов (Object Tracking) и связывание кадров.
*   Используемый метод: Интегрированный метод трекинга в YOLOv11: `model.track(source, tracker="bytetrack.yaml", persist=True)`.
*   Алгоритмическая суть: Метод использует фильтр Калмана (Kalman Filter) для прогнозирования траектории движения автомобиля на основе его предыдущей скорости и алгоритм Венгерского метода (Hungarian algorithm) для сопоставления предсказанных рамок с новыми детекциями. Параметр `persist=True` заставляет систему сохранять ID автомобиля при его временном перекрытии другими объектами.

### 2.4. Streamlit (streamlit)
Роль в проекте: Построение веб-интерфейса и дашборда аналитики.
*   Используемые методы:
    *   `st.image()` или `st.video()` — вывод обработанного видеопотока с наложенной графикой CV в режиме реального времени.
    *   `st.line_chart()` / `st.bar_chart()` — динамическое отображение графиков загруженности перекрестка (метрика: "количество авто в минуту").


## 3. Математические и алгоритмические методы анализа

Система опирается на три основных программных метода обработки метаданных:

### Метод 1: Определение вектора движения через историю трекинга
Для каждого уникального `track_id` система хранит массив из последних $N$ координат центра объекта $(x_c, y_c)$. 
*   Алгоритм: Сравнение начальной точки появления объекта на экране и конечной точки его исчезновения позволяет вычислить вектор и определить маневр (поворот налево, направо, движение прямо).

### Метод 2: Подсчет объектов в зоне интереса (ROI — Region of Interest)
На кадр накладывается невидимая маска (полигон полосы движения), заданная массивом точек.
*   Алгоритм: Нижняя центральная точка баундинг-бокса автомобиля передается в функцию `cv2.pointPolygonTest(polygon, point, False)`. Если метод возвращает `1` или `0`, объект находится внутри полосы и инкрементирует счетчик данной зоны.

### Метод 3: Расчет метрики «Длина очереди»
*   Алгоритм: Если объект классифицирован как транспортное средство, находится внутри ROI перед стоп-линией, а его вектор смещения за последние $K$ кадров равен нулю (или $\le \epsilon$), объект считается стоящим в очереди. Сумма таких объектов формирует метрику загрузки полосы для светофора.
